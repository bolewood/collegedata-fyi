# Yale University, CDS 2024-25 — known extraction issues

**Source PDF:** `yale_cds_2024-25_rmd_20250612.pdf`
**Docling extraction quality:** Good overall; mostly cosmetic structural quirks
**Last verified:** 2026-04-11

Yale's CDS extracted cleanly on the data side. All spot-checked numeric values match the source PDF, including B1 enrollment, B2 race/ethnicity, B3 degrees awarded, C7 factor-importance ratings, C9 test scores, H5 loan borrowing, and J disciplinary areas. Section hierarchy, checkbox states (☒/☐), hyperlinks, and multi-page table continuations all survived.

The issues below are structural rather than data-corrupting — consumers who read the JSON carefully will get correct answers, but naive table-grammar parsers may miss some fields.

## 1. Empty / sparse tables flattened to paragraphs

Yale left C11 (GPA distribution) and C12 (average HS GPA) blank. Docling rendered them as stacked one-line paragraphs instead of tables:

```
Score Range
Percent
Percent who had GPA of 4.0
Percent who had GPA between 3.75 and 3.99
...
```

Parsers that key off the table grammar will miss these fields entirely. Same issue affects the C14 "Application closing date" one-row table.

**Mitigation:** Post-processor should detect known CDS field labels and reconstruct the table shape even when cells are blank.

## 2. Single-cell boxed values promoted to H2 headings

Wherever the CDS puts an answer in its own 1×1 box, Docling emits it as a section header:

```
## 1665        (H4 graduate count)
## 489         (H6 nonresident aid count)
## $83,878.16  (H6 average aid dollars)
## $41,016,421 (H7 total aid dollars)
```

Values are correct but they lose their label context, and a naive heading-based section splitter will treat "1665" as a new section.

**Mitigation:** Reattach orphan heading-lines to the preceding label paragraph during normalization.

## 3. `nan` in empty cells

Empty cells in some tables render as the literal string `nan` (pandas default). Line 1343 of the extracted markdown has examples in the H3 non-need-based aid table.

**Mitigation:** Normalize `nan` → empty string in post-processing.

## 4. Duplicate header rows at page breaks

The H5 loan-source table emits an empty header-only table fragment right before the real table starts (where the page break lands). Cosmetic; parsers must tolerate a duplicate header.

## 5. Bulleted paragraphs absorbed into headings

Lines like `## · Do NOT include any aid related to the CARES Act…` happen where middle-dot and checkmark bullet characters were treated as heading markers.

**Mitigation:** Strip leading bullet glyphs from heading lines during post-processing.

## 6. Redundant checkbox encoding

Yes/No items come out as both a markdown checklist and the `☒`/`☐` glyph (`- [x] ☒ Yes`). Not wrong, just pick one during post-processing.

## Source-document quirks (not Docling bugs)

- "commiment" and "Undergra d" spellings in the body are in the source PDF, not introduced by extraction.
